# Инструкция по массовому импорту автомобилей

## Описание

Функционал массового импорта позволяет администраторам загружать данные о десятках автомобилей одним файлом вместо ручного добавления каждого по отдельности.

## Как использовать

### 1. Подготовка файла

Создайте текстовый файл (.txt) в формате TAB-разделённых значений:

```
Марка	Номер	Тип_пропуска	Дата
```

**Пример файла:**
```
Лада Приора	В 782 РХ 12	Постоянный	31.12.2026
КИА Серато	Н 122 АМ 73	Временный	15.06.2026
Фольксваген Пассат	С 999 РС 152	Постоянный	31.12.2026
BMW	В 460 РК 12	Временный	20.08.2026
```

### 2. Запуск импорта

**Через команду:**
1. Отправьте боту команду `/import`
2. Бот попросит отправить файл
3. Отправьте подготовленный .txt файл
4. Дождитесь отчёта о результатах

**Через меню:**
1. Откройте главное меню (`/start`)
2. Нажмите кнопку "📥 Импорт"
3. Отправьте подготовленный .txt файл
4. Дождитесь отчёта о результатах

### 3. Отчёт об импорте

После обработки файла бот отправит отчёт:

```
📊 Результаты импорта

✅ Добавлено: 25
⚠️ Пропущено (дубликаты): 3
❌ Ошибок: 1

Детали ошибок:
• Строка 15: Некорректный формат номера
```

## Формат данных

### Поля (разделяются TAB)

1. **Марка** — марка автомобиля (например: Лада Приора, BMW, КИА)
2. **Номер** — государственный номер (например: В 782 РХ 12)
3. **Тип пропуска** — "Постоянный" или "Временный"
4. **Дата** — дата окончания в формате ДД.ММ.ГГГГ (например: 31.12.2026)

### Правила

- **Разделитель:** Поля разделяются символом TAB (не пробелами!)
- **Тип пропуска:**
  - `Постоянный` → автомобиль получает постоянный доступ
  - `Временный` → автомобиль получает временный доступ до указанной даты
- **Дата:**
  - Для постоянных пропусков дата игнорируется
  - Для временных пропусков дата обязательна
  - Формат: ДД.ММ.ГГГГ (например: 31.12.2026)
  - Автоматически конвертируется в формат БД

### Обработка данных

- **Номера:** автоматически нормализуются (убираются пробелы, приводятся к верхнему регистру)
- **Статус доступа:** все импортируемые автомобили получают статус `allowed` (разрешён)
- **Дубликаты:** если номер уже есть в БД, запись пропускается (не обновляется)

## Ограничения

- Только администраторы могут импортировать данные
- Максимальный размер файла: 5 МБ
- Поддерживаемый формат: только .txt
- Кодировка: UTF-8

## Примеры ошибок

### Некорректный формат

❌ **Неправильно** (разделение пробелами):
```
Лада Приора В 782 РХ 12 Постоянный 31.12.2026
```

✅ **Правильно** (разделение TAB):
```
Лада Приора	В 782 РХ 12	Постоянный	31.12.2026
```

### Неправильный тип пропуска

❌ **Неправильно:**
```
Лада Приора	В 782 РХ 12	Долгосрочный	31.12.2026
```

✅ **Правильно:**
```
Лада Приора	В 782 РХ 12	Постоянный	31.12.2026
```

### Отсутствие даты для временного пропуска

❌ **Неправильно:**
```
КИА Серато	Н 122 АМ 73	Временный	
```

✅ **Правильно:**
```
КИА Серато	Н 122 АМ 73	Временный	15.06.2026
```

## Технические детали

### Архитектура

- **ImportService** — парсинг файла, валидация, конвертация данных
- **DocumentHandler** — обработка загруженных файлов
- **ImportStateHandler** — управление состоянием ожидания файла
- **TelegramApi** — методы getFile() и downloadFile() для работы с файлами

### Безопасность

- ✅ Проверка прав администратора
- ✅ Валидация типа файла
- ✅ Валидация каждой строки
- ✅ Защита от SQL-инъекций (prepared statements)
- ✅ Нормализация номеров
- ✅ Не перезаписывает существующие записи

## Отмена операции

В любой момент можно отменить импорт:
- Отправьте команду `/cancel`
- Или нажмите кнопку "❌ Отмена"
