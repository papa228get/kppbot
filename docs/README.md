# Telegram Бот для КПП

Telegram-бот для контроля доступа автомобилей на КПП с современной модульной архитектурой.

## Возможности

- ✅ Проверка номера автомобиля по базе данных
- 🔐 Управление доступом (разрешен/запрещен)
- 📅 Поддержка временных и постоянных пропусков
- ⏰ Автоматическая проверка срока действия пропуска
- 👨‍💼 Административные команды для управления базой
- 📥 **Массовый импорт автомобилей из файла**
- 💾 Легковесная SQLite база данных
- 🏗️ Модульная архитектура с namespaces

## Требования

- PHP 7.4 или выше
- Расширение SQLite3 (обычно включено по умолчанию)
- Бесплатный хостинг (например, InfinityFree)
- Telegram бот токен от [@BotFather](https://t.me/BotFather)

## Установка

### 1. Создайте Telegram бота

1. Откройте [@BotFather](https://t.me/BotFather) в Telegram
2. Отправьте команду `/newbot`
3. Следуйте инструкциям и получите токен бота
4. Сохраните токен - он понадобится позже

### 2. Получите ваш Telegram ID

1. Откройте [@userinfobot](https://t.me/userinfobot)
2. Отправьте любое сообщение
3. Бот вернет ваш ID - сохраните его

### 3. Загрузите файлы на хостинг

1. Зарегистрируйтесь на [InfinityFree](https://infinityfree.com)
2. Создайте новый сайт
3. Загрузите все файлы проекта в папку `htdocs` через File Manager или FTP

### 4. Настройте конфигурацию

Откройте файл `config.php` и измените:

```php
// Вставьте токен вашего бота
define('BOT_TOKEN', 'YOUR_BOT_TOKEN_HERE');

// Вставьте ваш Telegram ID (можно несколько через запятую)
define('ADMIN_IDS', [123456789, 987654321]);
```

### 5. Установите webhook

1. Откройте файл `setWebhook.php` в браузере
2. Измените URL на ваш домен:
   ```php
   $webhookUrl = "https://your-domain.infinityfreeapp.com/webhook.php";
   ```
3. Откройте в браузере: `https://your-domain.infinityfreeapp.com/setWebhook.php`
4. Вы должны увидеть сообщение "✅ Webhook успешно установлен!"

### 6. Готово!

Откройте вашего бота в Telegram и отправьте `/start`

## Использование

### Для охранников (обычные пользователи)

Просто отправьте номер автомобиля боту:
```
А123БВ
```

Бот ответит:
- ✅ **ПРОПУСТИТЬ** - если автомобиль разрешен
- ⛔ **НЕ ПРОПУСКАТЬ** - если запрещен
- ❌ **Автомобиль не найден** - если номера нет в базе

### Для администраторов

Используйте кнопки главного меню:

- **📋 Список** - просмотр всех автомобилей с пагинацией и поиском
- **📥 Импорт** - массовый импорт автомобилей из файла
- **📊 Статистика** - статистика по базе данных
- **ℹ️ Справка** - справка по использованию

#### Команды

- `/start` - Главное меню
- `/help` - Справка
- `/import` - Массовый импорт из файла
- `/remove НОМЕР` - Удалить автомобиль
- `/list` - Показать все автомобили
- `/stats` - Показать статистику
- `/cancel` - Отменить текущую операцию

#### Массовый импорт

Для загрузки большого количества автомобилей используйте команду `/import` или кнопку "📥 Импорт":

1. Подготовьте текстовый файл (.txt) в формате TAB-разделённых значений
2. Отправьте команду `/import` или нажмите кнопку "📥 Импорт"
3. Отправьте файл боту
4. Получите отчёт о результатах импорта

**Формат файла:**
```
Марка	Номер	Тип_пропуска	Дата
Лада Приора	В 782 РХ 12	Постоянный	31.12.2026
КИА Серато	Н 122 АМ 73	Временный	15.06.2026
```

📖 Подробная инструкция: [docs/IMPORT_GUIDE.md](IMPORT_GUIDE.md)

1. **Разделение ответственности** - каждый класс отвечает за одну задачу

3. **Масштабируемость** - легко добавлять новые функции
4. **Читаемость** - понятная структура папок и файлов
5. **Переиспользование** - сервисы и форматтеры можно использовать в разных местах
6. **Современный PHP** - использование namespaces и autoloading

## Структура базы данных

База данных SQLite создается автоматически при первом запуске.

Таблица `vehicles`:
- `id` - уникальный идентификатор
- `plate_number` - номер автомобиля (уникальный)
- `brand` - марка автомобиля
- `access_status` - статус доступа (allowed/denied)
- `pass_type` - тип пропуска (permanent/temporary)
- `expiry_date` - дата окончания пропуска
- `notes` - дополнительные заметки
- `created_at` - дата добавления

## Безопасность

- ✅ Только администраторы могут управлять базой данных
- ✅ ID администраторов хранятся в конфигурации
- ✅ База данных SQLite защищена от прямого доступа
- ✅ Валидация и нормализация всех входных данных
- ⚠️ Рекомендуется добавить `.htaccess` для защиты файлов конфигурации

### Рекомендуемый .htaccess

Создайте файл `.htaccess` в корневой папке:

```apache
# Запретить доступ к конфигурации и базе данных
<FilesMatch "^(config\.php|kpp\.db)$">
    Order allow,deny
    Deny from all
</FilesMatch>

# Разрешить доступ только к webhook.php и setWebhook.php
<FilesMatch "^(webhook|setWebhook)\.php$">
    Order allow,deny
    Allow from all
</FilesMatch>
```

## Разработка

### Добавление нового функционала

1. **Новый сервис** - создайте класс в `services/` с namespace `Services`
2. **Новый форматтер** - создайте класс в `formatters/` с namespace `Formatters`
3. **Новая команда** - добавьте метод в `CommandHandler`
4. **Новый callback** - добавьте обработку в `CallbackHandler`

### Autoloading

Проект использует PSR-4 autoloader в `bootstrap.php`. Все классы загружаются автоматически по namespace.

## Устранение неполадок

### Бот не отвечает
1. Проверьте, что webhook установлен: откройте `setWebhook.php` в браузере
2. Проверьте токен бота в `config.php`
3. Убедитесь, что файлы загружены в правильную папку (`htdocs`)

### Ошибка "Database locked"
- InfinityFree может иметь ограничения на одновременные запросы к SQLite
- Обычно это временная проблема, попробуйте еще раз

### Команды администратора не работают
- Проверьте, что ваш Telegram ID добавлен в `ADMIN_IDS` в `config.php`
- Убедитесь, что ID указан как число, без кавычек
